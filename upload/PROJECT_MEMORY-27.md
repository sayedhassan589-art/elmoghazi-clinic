## 5. نقاط مهمة للتعديلات المستقبلية

### 5.1 مشاكل تم حلها
| المشكلة | الحل |
|---------|------|
| **أسماء المرضى تظهر كأكواد** | كانت أرقام الملفات (F-MOLSSZLN) تظهر بدل الأسماء العربية بسبب تصميم البطاقة. تم إصلاح التصميم وتطبيق safeName() في كل الأماكن |
| **خطأ reminders is not defined** | fetchReminders كانت تستدعى بدون params. تم إصلاحها |
| **صور المرضى رمادية** | كان API بيرجع بدون base64 data. تم إصلاحه |
| **تقريب على iOS** | تم إضافة 16px كحد أدنى لخطوط الإدخال |

### 5.2 نقاط يجب الانتباه لها عند أي تعديل
1. **page.tsx ملف ضخم** - أي تعديل لازم يتبع نفس النمط الموجود
2. **كل الحالة في Home()** - State declarations كلها فوق (حوالي سطر 400-600)
3. **Callback functions** كلها في النص (حوالي سطر 1100-1400)
4. **المكونات الفرعية** معرفة في آخر الملف (بعد سطر 5000)
5. **دالة safeName()** (سطر ~250) - لازم تستخدمها في كل عرض لاسم مريض
6. **API calls** كلها في `src/lib/api.ts` - أضف الدالة الجديدة هناك أولاً
7. **قاعدة البيانات** - أي model جديد يضاف في `prisma/schema.prisma` ثم `prisma db push`

### 5.3 الاتفاقيات المستخدمة
- **أسماء المتغيرات**: camelCase
- **أسماء المكونات**: PascalCase
- **النصوص في الواجهة**: كلها عربية
- **الألوان**: نظام classes من Tailwind (bg-emerald-500, text-rose-600, إلخ)
- **الحجم النصي**: text-xs, text-sm, text-base, text-lg
- **الأيقونات**: كلها من lucide-react
- **Toast**: نستخدم `toast` من `sonner` (مش useToast)
- **API pattern**: `fetchAPI<T>(path, options)` في api.ts

## 7. التعليمات السريعة

### لإضافة model جديد في قاعدة البيانات:
1. أضف النموذج في `prisma/schema.prisma`
2. شغّل `npx prisma db push`
3. أضف API route في `src/app/api/الاسم/route.ts`
4. أضف API functions في `src/lib/api.ts`
5. أضف الـ state في `Home()` في `page.tsx`
6. أضف الـ callbacks في `page.tsx`
7. أضف/عدّل المكون المناسب في `page.tsx`
